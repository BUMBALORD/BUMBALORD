What is Pair Programming?

Pair programming refers to when 2 programmers work together to build a code based project . It entails one person serving as the role of a driver, and the other as the navigator. Pair programming is important because it helps minimize human error and maximize human creativity.

Why is Pair Programming important to DBC?

Pair programming to core to DBC's curriculum for several reasons. The first reason is that it helps reiterate previously learned concepts. Secondly, it gives 2 people a chance to practice vocalizing their ideas on building a code based project together. Lastly, it simulates the industry standard of building code based project as a team.

What is the difference between Driver and Navigator?

The Navigator is the brain and the Driver is the hands. They both work together collaboratively. Usually, they switch roles throughout the project in order to both see the project from each perspective.
