Arrays and Hashes

Jan 31st, 2015

Wrong type of Array!


Arrays in Ruby are a means of keeping track of any number of objects in an organized and easy to use fashion. In a simple way to put it, they are essentially a list of strings or integers, or really anything you want, that can store each element you wish and then be referenced back to when you need. Arrays generally look a lot like this:


countries = ['America', 'China', 'Turkmenistan']


Each object in the array is then given a number that can reference it later down the line when we want to get the information out of the array. It's as simple as counting to find out what your array element's number is. However, the first object in the array does not start at 1, it starts at 0. So, for the example above, We would have:


0 (for America), 1 (for China), and 2 (for Turkmenistan)


If we wanted to call on Turkmenistan within the array, we would simply call the method and place whatever number it is within a bracket like so:


countries[2]



Wrong type of Hash!


Hashes are a lot like arrays, only they don't have to be in a row, and they don't have to be referenced with a number. They can be referenced by whatever it is you wish to reference them with. This is because they are made up a collection of Keys and Value Pairs that allow you to sort objects within them for easy access later. Hashes look a lot like this, usually:


my_hash = {

"name" => "David"

"age" => 32

"birthplace" => "Oakland"

}


In this example, the first column is the key, while the second column is the value. "David" is the value for the key "name." This is called "literal notation." Within a hash, you cannot have duplicate keys as they will be overwritten with the newest value assigned to it. To access information inside the hash, we would call it in a similar way to an array, only we would use the key to call it. If we wanted to get "David" out of the hash, we would write:


my_hash["name"]


We can also call upon hashes with different methods. If the method, "each," speaks to every value in the hash, we could write something like this to have every value returned to us at once:


my_hash.each do |key, value|

puts value

end


This would return us every value in the hash, "my_hash," as we've defined the keys and values inside the hash and asked it to return each value to us. In this case, we would get:


David

32

Oakland
